Implementation Notes

Overview of Bootstrap components, built-in behaviours, custom JavaScript and hosting used in the Interactive Portfolio Website.

The portfolio is a single-page site built on Bootstrap 5.3.3 with a custom green theme. This page focuses on:

  • Bootstrap features used, including components and built-in behaviours.
  • Custom JavaScript features that add interactivity beyond Bootstrap.
  • Hosting details.

Main file: index.html · Custom styles: css/style.css · Custom scripts: js/scripts.js

1. Bootstrap components and features used

This section lists Bootstrap features that provide layout, components and behaviour without custom implementation.

1.1 Layout and structure

  • Grid system and containers: .container, .row, .col-*, and responsive classes such as .col-lg-5 and .row-cols-1 row-cols-md-3 control the layout in the hero, about, education, projects, experience, skills, hobbies and contact sections.
    Docs: Containers, Grid
  • Utility classes: spacing classes such as .mb-4 and .pt-4, flex utilities such as .d-flex, .justify-content-between, .align-items-center, text utilities and color helpers are used for alignment and spacing.
    Docs: Flex utilities, Spacing, Text
  • Responsive images: classes such as .img-fluid, .d-block, .w-100 and .rounded-3 are used in the hero portrait and hobbies gallery to keep images responsive and well framed.
    Docs: Images

1.2 Navigation and page behaviour

  • Navbar: the top navigation uses .navbar, .navbar-expand-lg, .navbar-dark and .bg-dark with a brand and section links. It collapses into a hamburger menu on smaller screens.
    Used at the top of index.html.
    Docs: Navbar
  • Collapse: the mobile menu uses Collapse via data-bs-toggle="collapse" and data-bs-target="#navbarContent". Bootstrap handles show and hide behaviour.
    Docs: Collapse
  • Scrollspy: enabled on <body> with data-bs-spy="scroll" and data-bs-target="#mainNav". Bootstrap updates the active navbar link based on the scroll position.
    Docs: Scrollspy

1.3 Content and data display

  • Card: used in multiple sections, including About (quick facts), Projects, Experience, Skills, Achievements, Hobbies and Contact, to group related content.
    Docs: Card
  • List group: used to present quick facts in About and contact items in the Quick Contact modal.
    Docs: List group
  • Table: the Education section uses .table, .table-striped, .table-hover, .align-middle and .table-responsive to display modules, semester, type and grade.
    Docs: Tables
  • Badge: Bootstrap badges are used with custom colors to label skills, project categories and achievements, for example in About, Projects and Achievements.
    Docs: Badges

1.4 Navs, tabs, pills and accordion

  • Navs and tabs: the Education section uses .nav, .nav-tabs and data-bs-toggle="tab" to switch between Year 1, 2, 3 and 4. Tab behaviour is provided by Bootstrap's JavaScript.
    Docs: Navs and tabs
  • Nav pills: the Skills section uses .nav-pills to switch between Technical skills and Soft skills.
    Docs: Nav pills
  • Accordion: the Experience section is a Bootstrap accordion built from .accordion, .accordion-item, .accordion-button and .accordion-collapse. Opening and closing is handled by Bootstrap.
    Docs: Accordion
  • Dropdown: the project filter in the Projects section uses a Bootstrap dropdown with .dropdown, .dropdown-toggle, .dropdown-menu and .dropdown-item. Bootstrap handles menu positioning and toggling.
    Docs: Dropdowns

1.5 Overlays and feedback

  • Modal: used in several places:
    • Project detail modals in the Projects section.
    • Email draft modal in the Contact section.
    • Quick Contact modal at the bottom of the page.
    • Gallery modal in the Hobbies section.
    Markup follows the Bootstrap modal pattern with data-bs-toggle="modal" and data-bs-target. Opening, closing and focus management come from Bootstrap.
    Docs: Modal
  • Carousel: the Hobbies section uses a Bootstrap carousel for featured photos, with .carousel, .carousel-inner, .carousel-item and previous or next controls. Sliding behaviour is built in.
    Docs: Carousel
  • Toast: a toast appears when the user submits the email form. It uses .toast with a close button and is shown programmatically.
    Docs: Toasts
  • Close button: .btn-close is used in modals and on the toast for consistent accessible close icons.
    Docs: Close button

1.6 Forms, buttons and icons

  • Forms: the email form inside the Contact modal uses Bootstrap form markup, including .form-label, .form-control, .invalid-feedback and works with Bootstrap validation states (via .was-validated added in JavaScript).
    Docs: Forms
  • Buttons: the site uses .btn, .btn-primary, .btn-outline-secondary, .btn-sm, .btn-lg and full width buttons such as .btn w-100 in the Contact section.
    Docs: Buttons
  • Bootstrap Icons: icons for LinkedIn, GitHub, email and navigation come from the Bootstrap Icons library loaded from the CDN.
    Docs: Bootstrap Icons

2. Custom dynamic JavaScript features

The following behaviours are implemented in js/scripts.js and build on top of Bootstrap components.

  • Scrollspy initialisation: the script explicitly initialises bootstrap.ScrollSpy on document.body with target: "#mainNav" to ensure consistent behaviour when scrolling through sections.
  • Project filtering: the Bootstrap dropdown in the Projects section is the control, but the filtering logic is custom. When a dropdown item is clicked, JavaScript:
    • Reads the selected filter from data-filter.
    • Shows or hides project cards based on their data-category attribute.
    • Updates the label on the dropdown toggle button.
  • Email draft and toast: in the Contact section, the email form:
    • Uses client side validation to check required fields and email format.
    • Builds a mailto: link using name, subject and message from the form.
    • Shows a Bootstrap toast with the message that the mail app is opening.
    • Triggers the default mail application using window.location.href.
  • Gallery modal image swapping: all thumbnails in the Hobbies gallery open the same Bootstrap modal. JavaScript listens for clicks on elements with the class .gallery-image, reads the src and alt, and updates #galleryModalImage so the correct photo appears in the modal.
  • Dynamic year in the footer: the current year is inserted into the footer span with id year on the main page and impl-year on this page using JavaScript.
